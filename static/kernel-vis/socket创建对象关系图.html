Socket 创建对象关系图

socket(AF_INET, SOCK_STREAM, 0) · Linux 5.15.78 · 从 current 出发追踪到协议栈 · 悬停对象查看源码级详情

per-cpu 变量，指向当前 CPU 上运行的 task_struct 定义: arch/x86/include/asm/current.h DECLARE_PER_CPU(struct task_struct *, current_task); static __always_inline struct task_struct *get_current(void) { return this_cpu_read_stable(current_task); } #define current get_current() 每个 CPU 核独立一份；调度切换时更新。 socket() 系统调用在内核态通过 current 访问当前进程的 files/nsproxy/signal 等资源。
每个线程/进程一个 task_struct（task 和 thread 同结构） 关键字段: pid / tgid 线程号 / 进程号 files → files_struct（CLONE_FILES 共享） nsproxy → net_ns（网络命名空间） signal->rlimit RLIMIT_NOFILE 等软硬限制 cred 权限/LSM 上下文 socket() 调用路径: __sys_socket 通过 current->nsproxy->net_ns 取网络命名空间 通过 current->files 写 fd 表
每个进程/线程默认独立，clone(CLONE_FILES) 时共享 关键字段: fdt 当前活跃 fdtable（初始指向内嵌 fdtab） fdtab 内嵌 fdtable（64 槽，避免短命进程 kmalloc） next_fd 启发式扫描起点，避免每次从 0 开始 file_lock spinlock，保护位图/数组/扩容 count 引用计数（共享 > 1） resize_in_progress 扩容互斥标志 resize_wait 并发扩容等待队列 扩容时 synchronize_rcu 等待所有 RCU 读者退出， 然后 copy_fdtable + rcu_assign_pointer 切换 fdt 指针， 旧表 call_rcu 延迟释放。
管理一个进程所有 fd 的索引结构 关键字段: max_fds 当前容量（64/128/256... 2 幂增长） fd[] file 指针数组（kvmalloc_array） open_fds 一级占用位图（每 bit 一个 fd） close_on_exec exec 时自动关闭的位图 full_fds_bits 二级摘要：每 bit 概括 64 fd 是否全满 内嵌初始化（files_struct.fdtab）: fd → files->fd_array open_fds → files->open_fds_init close_on_exec → files->close_on_exec_init full_fds_bits → files->full_fds_bits_init 扩容后 fdt 指向 kmalloc 的新表，三块位图一次 kvmalloc 首尾相接。
进程看到的文件描述符（整数索引） fdt->fd[fd] 指向 struct file 上界受 RLIMIT_NOFILE + sysctl_nr_open 双重限制 alloc_fd() 两级位图查找: full_fds_bits → find_next_zero_bit（跳过全满 64-fd 块） open_fds[块] → find_next_zero_bit（精确命中空闲位） 分配与安装分离: alloc_fd 仅置位图，fdt->fd[fd] 仍为 NULL fd_install rcu_assign_pointer 写入 file 指针 中途失败 put_unused_fd 归还位图，不产生悬挂 fd
alloc_file_pseudo() 分配，file slab 管理 socket_file_ops 定义: read_iter → sock_read_iter write_iter → sock_write_iter poll → sock_poll release → sock_close mmap → sock_mmap stream_open() 标记 FMODE_STREAM file->private_data = socket（双向关联）
d_alloc_pseudo() 创建 挂在 sock_mnt 下 dname 取自协议名，如 "TCP" d_inode 指向同一 socket_alloc 中的 vfs_inode
嵌在 socket_alloc 偏移 0 处 与 vfs_inode 一体分配（同一 slab 对象） inet_stream_ops 定义: bind → inet_bind connect → inet_stream_connect listen → inet_listen accept → inet_accept sendmsg → inet_sendmsg recvmsg → inet_recvmsg poll → tcp_poll sock_alloc_inode() 从 sock_inode_cachep 分配
嵌在 socket_alloc 紧接 socket 之后 i_mode = S_IFSOCK | S_IRWXUGO i_op = sockfs_inode_ops i_ino = get_next_ino()（递增） SOCKET_I(inode): container_of → socket_alloc → .socket SOCK_INODE(socket): container_of → socket_alloc → .vfs_inode 零开销：编译期偏移量，运行时仅指针减法
sk_alloc() 从 tcp_prot.slab 分配 (~2KB) 继承链: tcp_sock └ inet_connection_sock (ICSK 定时器) └ inet_sock (IP 地址/端口) └ sock (通用传输层) └ sock_common (最小公共) tcp_init_sock() 初始化: 重传/延迟ACK/keepalive 三个定时器 初始 RTO = 1s，最小 RTO = 200ms 初始 cwnd = 10 (RFC 6928) ssthresh = 0x7fffffff 默认拥塞控制: CUBIC sock_init_data() 建立双向关联: sk->sk_socket = socket socket->sk = sk
