# Linux 5.15.78 内核源码架构分析

## 一、整体架构分层

Linux 内核采用**宏内核(Monolithic)架构**，所有核心服务运行在同一特权级(Ring 0)的同一地址空间，通过函数调用而非 IPC 通信，兼具高性能和模块可加载的灵活性。

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户空间 (Ring 3)                         │
│    应用程序  │  glibc  │  init/systemd  │  shell  │  daemon     │
├─────────────────────── 系统调用接口 ────────────────────────────┤
│                        内核空间 (Ring 0)                         │
│                                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │ 进程管理  │ │ 内存管理  │ │ 文件系统  │ │ 网络协议栈│           │
│  │ (kernel/) │ │  (mm/)   │ │  (fs/)   │ │  (net/)  │           │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └────┬─────┘           │
│       │            │            │            │                  │
│  ┌────┴────────────┴────────────┴────────────┴──────┐           │
│  │          设备驱动框架 (drivers/)                    │           │
│  └──────────────────────┬───────────────────────────┘           │
│                         │                                       │
│  ┌──────────────────────┴───────────────────────────┐           │
│  │      体系结构相关层 (arch/x86, arch/arm64, ...)    │           │
│  └──────────────────────────────────────────────────┘           │
├─────────────────────────────────────────────────────────────────┤
│                        硬件 (CPU / 内存 / 外设)                  │
└─────────────────────────────────────────────────────────────────┘
```

## 二、目录结构与子系统对应关系

| 目录 | 子系统 | 核心职责 |
|------|--------|---------|
| `arch/` | 体系结构 | CPU 引导、页表、中断入口、系统调用表、上下文切换汇编 |
| `kernel/` | 进程管理 | 调度器、fork、信号、workqueue、RCU、锁、cgroup |
| `mm/` | 内存管理 | 伙伴系统、slab、vmalloc、mmap、缺页处理、页回收 |
| `fs/` | 文件系统 | VFS 框架、具体文件系统(ext4/proc/sysfs)、块层接口 |
| `net/` | 网络 | socket 接口、IPv4/IPv6、TCP/UDP、Netfilter、驱动接口 |
| `drivers/` | 设备驱动 | 字符/块/网络/平台设备、总线(PCI/USB/I2C)、Android Binder |
| `include/` | 头文件 | 内核 API 和数据结构定义（子目录对应各子系统） |
| `init/` | 启动 | `start_kernel()`、initcall 机制、PID 1 创建 |
| `ipc/` | 进程间通信 | System V IPC (共享内存/信号量/消息队列) |
| `security/` | 安全 | LSM 框架、SELinux、AppArmor |
| `crypto/` | 密码学 | 内核加密 API、算法实现 |
| `lib/` | 通用库 | 数据结构(rbtree/list/xarray)、字符串、压缩 |
| `block/` | 块设备层 | I/O 调度、bio/request、多队列(blk-mq) |
| `sound/` | 音频 | ALSA 框架 |
| `tools/` | 工具 | perf、bpftool 等用户态工具 |
| `scripts/` | 构建脚本 | Kconfig、Kbuild、模块签名 |

## 三、内核启动全流程

### 3.1 从上电到 C 语言入口

```
电源 → CPU 复位
  │
  ├─ x86_64 路径:
  │    BIOS/UEFI → GRUB → 加载 bzImage
  │    → arch/x86/boot/header.S (实模式 16 位)
  │    → arch/x86/boot/compressed/head_64.S (解压内核)
  │    → arch/x86/kernel/head_64.S: startup_64 (恒等映射、GDT、栈)
  │    → arch/x86/kernel/head64.c: x86_64_start_kernel()
  │    → start_kernel()
  │
  └─ ARM64 路径:
       固件/引导加载器 → x0 = FDT 地址
       → arch/arm64/kernel/head.S: primary_entry
       → 建页表 → __primary_switch → __primary_switched
       → 切 init_task 栈、清 BSS
       → bl start_kernel
```

### 3.2 start_kernel() — 子系统初始化精确顺序

从 `init/main.c` 第 956 行开始，初始化严格分为 5 个阶段：

```
═══════════════════════════════════════════════════════════════
阶段 1: 早期初始化（中断禁止，单 CPU，无内存分配器）
═══════════════════════════════════════════════════════════════
  set_task_stack_end_magic()    栈溢出魔数
  smp_setup_processor_id()     Boot CPU ID
  cgroup_init_early()          cgroup 根初始化
  local_irq_disable()          ← 关中断
  boot_cpu_init()              CPU 位掩码
  setup_arch()                 ★ 体系结构初始化（设备树/内存布局/命令行）
  setup_per_cpu_areas()        Per-CPU 变量分配
  smp_prepare_boot_cpu()       Boot CPU 体系结构初始化

═══════════════════════════════════════════════════════════════
阶段 2: 内存子系统初始化
═══════════════════════════════════════════════════════════════
  build_all_zonelists()        ★ 建立 zone 空闲页链表
  page_alloc_init()            页分配器 per-CPU 缓存
  vfs_caches_init_early()      dcache / inode 哈希表（早期）
  mm_init()                    ★ slab / vmalloc / mem_init / kmem_cache

═══════════════════════════════════════════════════════════════
阶段 3: 调度器、中断、定时器
═══════════════════════════════════════════════════════════════
  sched_init()                 ★ 调度器（CFS/RT/DL 运行队列）
  workqueue_init_early()       工作队列框架
  rcu_init()                   ★ RCU 机制
  early_irq_init()             IRQ 描述符
  init_IRQ()                   ★ 中断控制器
  tick_init()                  时钟 tick
  init_timers()                低精度定时器
  hrtimers_init()              高精度定时器
  softirq_init()               ★ 软中断（网络/块 I/O 的基础）
  timekeeping_init()           时间管理
  time_init()                  体系结构时间源
  local_irq_enable()           ← 开中断（从此可响应中断）
  console_init()               ★ 控制台（从此可打印输出）

═══════════════════════════════════════════════════════════════
阶段 4: 各子系统初始化
═══════════════════════════════════════════════════════════════
  pid_idr_init()               PID 分配器
  fork_init()                  ★ 进程创建（task_struct slab / 线程数上限）
  proc_caches_init()           进程相关 slab 缓存
  cred_init()                  凭证子系统
  security_init()              ★ LSM 安全框架
  net_ns_init()                ★ 网络命名空间（网络栈基础）
  vfs_caches_init()            ★ VFS 完整初始化（super_block/inode/dentry/file）
  signals_init()               信号子系统
  proc_root_init()             /proc 文件系统
  cgroup_init()                cgroup 完整初始化
  check_bugs()                 CPU bug 检测与 workaround

═══════════════════════════════════════════════════════════════
阶段 5: 启动 init 进程
═══════════════════════════════════════════════════════════════
  arch_call_rest_init()
    → rest_init()
        → kernel_thread(kernel_init)    创建 PID 1
        → kernel_thread(kthreadd)       创建 PID 2
        → cpu_startup_entry()           PID 0 进入 idle
```

### 3.3 rest_init() → 用户空间

```
rest_init() [PID 0 上下文]
  │
  ├── kernel_thread(kernel_init)  ──→  PID 1
  │     │
  │     ├── kernel_init_freeable()
  │     │     ├── smp_init()           启动所有 secondary CPU
  │     │     ├── sched_init_smp()     SMP 调度器完整初始化
  │     │     ├── do_basic_setup()
  │     │     │     ├── driver_init()  设备驱动框架初始化
  │     │     │     ├── do_ctors()     全局构造函数
  │     │     │     └── do_initcalls() ★ 执行全部 initcall (level 0-7)
  │     │     ├── 打开 /dev/console
  │     │     └── 挂载根文件系统
  │     │
  │     └── kernel_execve("/sbin/init")  ──→  用户态 init 进程
  │
  ├── kernel_thread(kthreadd)  ──→  PID 2 (所有内核线程的父线程)
  │
  └── cpu_startup_entry()  ──→  PID 0 成为 idle 进程
```

### 3.4 Initcall 层次体系

`do_initcalls()` 按段地址顺序执行 8 个级别的初始化函数：

| 级别 | 宏 | 段名 | 用途 | 典型注册者 |
|------|-----|------|------|-----------|
| 0 | `pure_initcall` | `.initcall0` | 纯变量初始化，不依赖任何子系统 | 少数底层设施 |
| 1 | `core_initcall` | `.initcall1` | 内核核心依赖 | 内存、中断、基础设施 |
| 2 | `postcore_initcall` | `.initcall2` | 依赖 core 的后续逻辑 | kobject、irq_domain |
| 3 | `arch_initcall` | `.initcall3` | 体系结构特定初始化 | IOMMU、ACPI、平台 |
| 4 | `subsys_initcall` | `.initcall4` | 大型子系统框架 | PCI、USB、I2C 总线 |
| 5 | `fs_initcall` | `.initcall5` | 文件系统/协议族注册 | ext4、inet_init()、proc |
| 6 | `device_initcall` | `.initcall6` | 设备驱动注册（`module_init()` 默认） | 各种驱动程序 |
| 7 | `late_initcall` | `.initcall7` | 启动尾声，依赖面最广 | 调优、可选特性 |

另有 `early_initcall` 在 SMP 之前由 `do_pre_smp_initcalls()` 单独执行。

## 四、六大核心子系统架构

### 4.1 进程管理（kernel/）

**核心数据结构 `struct task_struct`（约 700+ 字段）：**

```
struct task_struct {
    // ─── 调度相关 ───
    __state                     运行状态 (RUNNING/INTERRUPTIBLE/...)
    prio / static_prio          优先级
    sched_class                 调度类指针 (stop/dl/rt/fair/idle)
    se (sched_entity)           CFS 调度实体 (含 vruntime / rb_node)
    rt (sched_rt_entity)        RT 调度实体
    on_rq                       是否在运行队列

    // ─── 内存 ───
    mm / active_mm              用户地址空间 / 活跃mm(内核线程借用)

    // ─── 文件系统 ───
    fs (fs_struct)              当前目录/根目录/umask
    files (files_struct)        打开文件表

    // ─── 信号 ───
    signal (signal_struct)      线程组共享信号状态
    sighand                     信号处理函数表
    pending                     私有待处理信号队列

    // ─── 标识 ───
    pid                         线程 ID
    tgid                        线程组 ID（用户态看到的"进程 ID"）
    stack                       内核栈指针
};
```

**进程创建流程：**

```
用户 fork() / clone() / vfork()
  → sys_clone / sys_clone3
    → kernel_clone()
      → copy_process()
      │   ├── dup_task_struct()      复制 task_struct + 分配内核栈
      │   ├── copy_files()           共享或复制文件描述符表
      │   ├── copy_fs()              共享或复制 fs_struct
      │   ├── copy_mm()              ★ COW 复制地址空间
      │   ├── copy_sighand()         共享或复制信号处理表
      │   ├── copy_thread()          设置子进程内核栈和返回地址
      │   └── alloc_pid()            分配 PID
      └── wake_up_new_task()         将子进程放入调度器运行队列
```

**CFS 调度器工作原理：**

```
per-CPU 运行队列 (struct rq)
  └── cfs_rq
        └── 红黑树 (按 vruntime 排序)
              ├── task A (vruntime=100)
              ├── task B (vruntime=120)  ← 最左节点 = 下一个运行
              └── task C (vruntime=150)

调度循环:
  schedule() → __schedule()
    → pick_next_task()
      → pick_next_task_fair()
        → pick_next_entity()    取 vruntime 最小的实体
    → context_switch()
        ├── switch_mm()         切换页表 (CR3/TTBR0)
        └── switch_to()         切换寄存器和栈（汇编）

每次运行时 update_curr():
  vruntime += delta_exec × (NICE_0_WEIGHT / task_weight)
  → 高权重任务 vruntime 增长慢，获得更多 CPU 时间
```

**进程状态转换：**

```
                 fork()
                   │
                   ▼
    ┌──────── TASK_RUNNING ◄─────── try_to_wake_up()
    │          (可运行/正在运行)              ▲
    │              │                        │
    │    schedule() / 阻塞                    │ 事件到达
    │              │                        │
    │              ▼                        │
    │    TASK_INTERRUPTIBLE ────信号────► TASK_RUNNING
    │    TASK_UNINTERRUPTIBLE ──完成───► TASK_RUNNING
    │
    │    do_exit()
    │       │
    │       ▼
    │   EXIT_ZOMBIE ──── wait() ──→ 回收
    │
    └──→ __TASK_STOPPED (SIGSTOP)
         __TASK_TRACED (ptrace)
```

### 4.2 内存管理（mm/）

**分层架构：**

```
用户空间 mmap()/brk()/malloc()
  │
  ▼
┌─────────────────────────────────────────┐
│  VMA 层: vm_area_struct + mm_struct      │  mm/mmap.c
│  管理进程虚拟地址空间 [vm_start, vm_end) │  延迟分配：创建 VMA，不分物理页
└───────────────────┬─────────────────────┘
                    │ 首次访问 → 缺页异常
                    ▼
┌─────────────────────────────────────────┐
│  缺页处理: handle_mm_fault()             │  mm/memory.c
│  → do_anonymous_page() (匿名页)          │  分配物理页 + 建立页表映射
│  → do_fault() (文件映射页)               │
│  → do_wp_page() (COW 写时复制)           │
│  → do_swap_page() (换入)                 │
└───────────────────┬─────────────────────┘
                    │ 需要物理页
                    ▼
┌─────────────────────────────────────────┐
│  Slab 层: kmalloc() / kmem_cache_alloc() │  mm/slub.c
│  小对象分配器 (per-CPU 快速路径)          │  对象级缓存
└───────────────────┬─────────────────────┘
                    │ 需要整页
                    ▼
┌─────────────────────────────────────────┐
│  伙伴系统: __alloc_pages()               │  mm/page_alloc.c
│  物理页帧分配器 (order 0~10)             │  get_page_from_freelist()
│  zone: DMA → DMA32 → NORMAL → HIGHMEM   │  → __alloc_pages_slowpath()
└───────────────────┬─────────────────────┘
                    │ 内存不足
                    ▼
┌─────────────────────────────────────────┐
│  页回收: kswapd / direct reclaim         │  mm/vmscan.c
│  shrink_node() → LRU 扫描 → 换出/丢弃    │  try_to_free_pages()
└─────────────────────────────────────────┘
```

**内存区域(Zone)划分：**

```
物理内存地址空间:
┌──────────────┬────────────────┬──────────────────┬─────────────┐
│  ZONE_DMA    │  ZONE_DMA32    │   ZONE_NORMAL    │ ZONE_HIGHMEM│
│  0~16MB      │  16MB~4GB      │   可直接映射区域  │ (仅32位)    │
│  ISA DMA     │  32位 DMA      │   内核正常使用    │             │
└──────────────┴────────────────┴──────────────────┴─────────────┘
```

### 4.3 文件系统（fs/）

**VFS 抽象层架构：**

```
用户: open("/etc/passwd", O_RDONLY)
  │
  ▼
┌────────────────────────────────────────────────────────┐
│  VFS 层 (fs/)                                          │
│                                                        │
│  struct file  ←───→  struct dentry  ←───→  struct inode│
│  (打开文件实例)      (目录项缓存)          (文件元数据)  │
│  f_op = ext4 ops     d_inode              i_op/i_fop   │
│  f_pos               d_parent/d_name      i_size       │
│                      d_sb                 i_mapping    │
│                           │                            │
│                      struct super_block                │
│                      (挂载的文件系统实例)               │
│                      s_op, s_type, s_root              │
└───────────────────────┬────────────────────────────────┘
                        │  调用具体文件系统的操作函数
                        ▼
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│   ext4   │ │  procfs  │ │  sysfs   │ │  tmpfs   │ ...
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

**路径查找流程（fs/namei.c）：**

```
filename_lookup()
  → path_lookupat()
    → path_init()           确定起点 (root / cwd / dirfd)
    → link_path_walk()      逐分量遍历
    │   对每个分量:
    │   ├── 权限检查
    │   ├── hash 计算
    │   ├── walk_component()  查 dcache → 未命中则 real_lookup → 调 i_op->lookup
    │   └── 处理挂载点穿越 / 符号链接跟随
    → lookup_last()         处理最终分量
    → complete_walk()       完成查找
```

**读写流程：**

```
read(fd, buf, count)
  → ksys_read() → vfs_read()
    → file->f_op->read()         直接读
       或 file->f_op->read_iter()  迭代器读
         → 具体文件系统实现
           → 页缓存命中？ 直接拷贝
           → 未命中？ 触发 readahead → 块 I/O → DMA → 拷贝到用户
```

### 4.4 网络协议栈（net/）

**分层架构：**

```
用户空间: socket() / bind() / connect() / send() / recv()
  │
  ▼
┌─────────────────────────────────────────────────────┐
│  Socket 层 (net/socket.c)                            │
│  struct socket (BSD 接口) → struct sock (协议控制块)  │
│  ops = inet_stream_ops / inet_dgram_ops              │
│  向上提供 socket/bind/connect/sendmsg/recvmsg 系统调用│
│  向下通过 proto_ops 分发到具体协议                     │
└───────────────────────┬─────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────┐
│  传输层 (net/ipv4/tcp*.c, udp.c)                     │
│  TCP: 连接管理/拥塞控制(BBR/CUBIC)/重传/流控           │
│  UDP: 无连接/校验和/多播                              │
└───────────────────────┬─────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────┐
│  网络层 / IP 层 (net/ipv4/)                           │
│  ┌─────────────────────────────────────────────────┐ │
│  │ Netfilter 钩子（嵌入在 IP 层内部，非独立模块）    │ │
│  │                                                 │ │
│  │ 收包: ip_rcv() ──[PREROUTING]──→ 路由判定        │ │
│  │          ├→ 本机: ──[INPUT]──→ 传输层             │ │
│  │          └→ 转发: ──[FORWARD]──→ [POSTROUTING]    │ │
│  │ 发包: 传输层 → ip_output() ──[OUTPUT]──→ 路由     │ │
│  │          → ──[POSTROUTING]──→ 出口                │ │
│  └─────────────────────────────────────────────────┘ │
│  路由查找: FIB rule → fib_table_lookup()              │
│  L3 master (VRF): l3mdev_update_flow() 自动选路由表   │
│  分片/重组: ip_fragment() / ip_defrag()               │
└───────────────────────┬─────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────┐
│  邻居子系统 (net/core/neighbour.c)                    │
│  ARP/NDP: IP → MAC 地址解析                           │
│  neigh_output() → 将 L3 包交给 L2 发送                │
└───────────────────────┬─────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────┐
│  设备子系统 (net/core/dev.c)                          │
│  struct net_device / struct sk_buff                  │
│  dev_queue_xmit() (发送) / netif_receive_skb() (接收)│
│  NAPI 收包 / Qdisc 发包 / TSO/GSO/GRO 卸载           │
└───────────────────────┬─────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────┐
│  网卡驱动 (drivers/net/)                              │
│  DMA ring buffer / 中断 / NAPI poll                  │
└───────────────────────┬─────────────────────────────┘
                        │
                   物理网口 (NIC)
```

**Netfilter 5 个钩子点的源码位置：**

| 钩子 | 源码位置 | 上下文 |
|------|---------|--------|
| `NF_INET_PRE_ROUTING` | `net/ipv4/ip_input.c:599` | `ip_rcv()` 尾部，路由判定之前 |
| `NF_INET_LOCAL_IN` | `net/ipv4/ip_input.c:307` | `ip_local_deliver()` 交给传输层之前 |
| `NF_INET_FORWARD` | `net/ipv4/ip_forward.c:157` | `ip_forward()` 转发判定完成后 |
| `NF_INET_LOCAL_OUT` | `net/ipv4/ip_output.c` | 本机发出包，路由查找之后 |
| `NF_INET_POST_ROUTING` | `net/ipv4/ip_output.c:643` | `ip_output()` 出口，发往驱动之前 |

**收包完整路径（含 VRF 介入点）：**

```
网卡 DMA → 硬中断 → NAPI poll → napi_gro_receive()
  → netif_receive_skb()
    → __netif_receive_skb_core()
      → packet_type 分发 (ip_packet_type → ip_rcv)
        → ip_rcv_core()         校验IP头, IPCB(skb)->iif = skb->skb_iif
        → NF_HOOK(PRE_ROUTING)
        → ip_rcv_finish()
          → l3mdev_ip_rcv()     ★ VRF 介入: skb->dev 改为 VRF master
          → ip_rcv_finish_core()
            → ip_route_input_noref()  路由查找
              → __fib_lookup()
                → l3mdev_update_flow()      iif 改为 VRF master
                → fib_rules_lookup()
                  → l3mdev_fib_rule_match() 自动选 VRF 路由表
          → 本机: ip_local_deliver() → NF_HOOK(INPUT) → tcp_v4_rcv()
          → 转发: ip_forward() → NF_HOOK(FORWARD) → ip_output()
```

**发包完整路径：**

```
send(fd, buf, len)
  → sock->ops->sendmsg()  (inet_sendmsg)
    → tcp_sendmsg() / udp_sendmsg()
      → ip_queue_xmit() / ip_send_skb()
        → __ip_queue_xmit()
          → ip_route_output_flow()  路由查找（出口方向）
            → __fib_lookup()
              → l3mdev_update_flow()  ★ oif 改为 VRF master
              → l3mdev_fib_rule_match() 选 VRF 路由表
          → NF_HOOK(OUTPUT)
        → ip_output()
          → NF_HOOK(POST_ROUTING)
            → ip_finish_output()
              → neigh_output()      邻居子系统: ARP 解析
                → dev_queue_xmit()  设备子系统: Qdisc 入队
                  → ndo_start_xmit()  驱动发送
                    → DMA → 物理网口
```

**inet_init() 注册机制（fs_initcall 级别）：**

inet_init()                               // net/ipv4/af_inet.c

### 4.5 设备驱动模型（drivers/）

**核心对象关系：**

```
struct bus_type (总线类型)
  ├── match()          匹配设备与驱动
  ├── probe()          绑定后初始化
  ├── devices klist    挂载的设备链表
  └── drivers klist    注册的驱动链表
       │                      │
       ▼                      ▼
struct device              struct device_driver
  ├── bus                    ├── bus
  ├── driver ──────────────→ ├── probe()
  ├── parent                 ├── remove()
  ├── kobj (sysfs)           └── of_match_table
  └── of_node (设备树)
```

**注册流程：**

```
设备注册:                          驱动注册:
device_register()                 driver_register()
  → device_initialize()             → bus_add_driver()
  → device_add()                      → driver_attach()
    → bus_add_device()                  → 遍历 bus 上所有设备
    → bus_probe_device()                → bus->match() 匹配
      → device_initial_probe()          → driver_probe_device()
        → 遍历 bus 上所有驱动              → really_probe()
        → bus->match()                     → driver->probe() 初始化
        → driver->probe()
```

## 六、三个特殊进程

| PID | 名称 | 创建方式 | 职责 |
|-----|------|---------|------|
| 0 | swapper/idle | 静态定义 `init_task`（init/init_task.c），非动态创建 | 执行 start_kernel() 全部初始化，然后永久进入 cpu_startup_entry() 的 idle 循环 |
| 1 | init | rest_init() 中 kernel_thread(kernel_init) | 执行 do_initcalls()，然后 kernel_execve("/sbin/init") 变身为用户态 init 进程，是所有用户进程的祖先 |
| 2 | kthreadd | rest_init() 中 kernel_thread(kthreadd) | 内核线程守护进程，所有内核线程（kworker/ksoftirqd/kswapd 等）的创建都由它代理完成 |
